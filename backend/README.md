## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

## Agentic AI Interview (MVP)

This API includes an agentic, stateful AI interview flow (Java/backend focused) powered by **OpenRouter free Qwen models only**.

### Environment

Required:

- `OPENROUTER_API_KEY`

Optional:

- `OPENROUTER_BASE_URL` (default: `https://openrouter.ai/api/v1`)
- `OPENROUTER_DEFAULT_MODEL` (default: `qwen/qwen2.5-7b-instruct:free`) — must end with `:free`

### Endpoints

All routes are under `GET/POST /api/v1/...`.

- `POST /campus/ai/interview/sessions` — start a new interview session and get the first question
- `POST /campus/ai/interview/sessions/:sessionId/answer` — submit an answer and get either a follow-up or the next question
- `POST /campus/ai/interview/sessions/:sessionId/end` — end the session and get final evaluation
- `GET /campus/ai/interview/sessions/:sessionId` — fetch state + transcript + evaluation (if ended)

### Example Java backend interview flow

1) Start session:

Request body:

```json
{
  "candidateId": "<uuid>",
  "roleTitle": "Java Backend Engineer",
  "seniority": "mid",
  "maxQuestions": 5,
  "maxFollowUps": 2
}
```

Response contains `nextPrompt` (first question).

2) Submit an answer:

Request body:

```json
{
  "answer": "<candidate answer>"
}
```

Response contains `nextPrompt`, which is either a depth follow-up (Why/tradeoffs/production failure modes) or the next adaptive question.

3) End session:

Response contains a structured evaluation:

```json
{
  "technicalDepthScore": 7,
  "problemSolvingScore": 6,
  "communicationScore": 8,
  "strengths": ["..."],
  "weaknesses": ["..."],
  "summary": "..."
}
```

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ npm install -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.
